# 首页功能实现计划

## 需求概述

### 功能需求
1. **需求输入区**：用户可在输入框中输入自己的设计需求
2. **生成按钮**：点击后跳转到 design 页面
3. **项目列表**：展示用户创建过的所有项目
4. **项目恢复**：点击项目可恢复完整状态（聊天记录、生成文件等）
5. **项目删除**：用户可删除不需要的项目
6. **数据持久化**：所有数据存储在浏览器 IndexedDB 中
7. **模型选择**：用户可在首页选择 AI 模型（从 Design 页面迁移）
8. **API Key 管理**：用户可在首页设置/查看 API Key（从 Design 页面迁移）
9. **生成前检查**：点击生成按钮前检查 API Key 是否设置，未设置则弹窗提示

### 非功能需求
- 页面要精美，不要太浓的AI味
- 使用 UI 组件库（shadcn/ui）
- 响应式设计，支持移动端和桌面端

## 设计方案

### 设计风格
- **风格名称**：Flat Design（扁平化设计）
- **设计理念**：简约、专业、现代，减少AI工具的"冰冷感"
- **配色方案**：
  - Primary: `#1E3A5F` (深海军蓝)
  - Secondary: `#2563EB` (蓝色)
  - Accent/CTA: `#059669` (绿色 - 用于主要操作)
  - Background: `#F8FAFC` (浅灰白)
  - Foreground: `#0F172A` (深灰)
  - Muted: `#F1F3F5` (灰色背景)
  - Border: `#E4E7EB` (边框色)
  - Destructive: `#DC2626` (红色 - 用于删除)
- **字体**：Inter（无衬线，清晰易读）
- **动效**：150-200ms 过渡动画，简洁不花哨

## 技术方案

### 技术栈
- **框架**：Next.js 16 + React 19
- **语言**：TypeScript
- **UI 组件库**：shadcn/ui
- **样式**：Tailwind CSS 4
- **存储**：idb（IndexedDB 封装库）
- **国际化**：next-intl（已有）

### 数据模型

```typescript
type Project = {
  id: string; // UUID
  title: string; // 项目标题（从需求中提取或默认标题）
  requirement: string; // 用户输入的需求文本
  createdAt: number; // 创建时间戳
  updatedAt: number; // 更新时间戳
  state: {
    messages: ChatMessage[]; // 聊天记录
    files: FileRecord[]; // 生成的文件
    activeFile: string | null; // 当前激活的文件路径
  };
};

type ChatMessage = {
  id: string;
  type: 'user' | 'assistant' | 'system' | 'tool-call' | 'tool-result' | 'done' | 'error';
  content: string;
  toolName?: string;
  toolArgs?: string;
  isStreaming?: boolean;
  timestamp: number;
};

type FileRecord = {
  path: string;
  content: string;
  createdAt: number;
};
```

### IndexedDB 结构
```
Database: snake-design-db
├── Store: projects
│   ├── keyPath: id
│   └── indexes:
│       ├── createdAt (用于排序)
│       └── updatedAt (用于排序)
```

## 实现步骤

### 阶段一：基础设施
1. **安装依赖**
   - 安装 `idb` (IndexedDB 封装)
   - 安装 shadcn/ui 及必要组件

2. **类型定义**
   - 创建 `Project` 类型定义
   - 创建相关辅助类型

3. **数据库层**
   - 初始化 idb
   - 实现 projects 数据库的 CRUD 操作

### 阶段二：首页组件
4. **HomeHeader 组件**
   - Logo 显示
   - 模型选择下拉框（从 Design 页面迁移）
   - API Key 按钮 + 状态指示器（已连接/未连接）
   - 点击 API Key 按钮打开 ApiKeyModal

5. **ApiKeyModal 组件**（从 Design 迁移）
   - 输入/编辑 API Key
   - 保存到 localStorage
   - 外部点击遮罩关闭

6. **HeroSection 组件**
   - 需求输入框（textarea）
   - 生成按钮
   - **生成前检查**：点击生成时检查 API Key，未设置则弹出 ApiKeyModal

7. **ProjectCard 组件**
   - 项目标题显示
   - 创建时间显示
   - 删除按钮（带确认对话框）
   - 点击跳转到设计页面

8. **ProjectList 组件**
   - 项目网格布局
   - 空状态显示
   - 加载骨架屏

9. **首页整合**
   - 组装各组件
   - 响应式布局

### 阶段三：与 Design 页面集成
10. **项目状态管理**
    - 设计页面的状态需要能从 Project.state 恢复
    - 退出设计页面时自动保存状态到 IndexedDB

11. **路由与参数**
    - 点击项目时传递 projectId
    - design 页面根据 projectId 加载项目状态

12. **Design 页面简化**
    - 移除 Design 页面原有的模型选择和 API Key 管理
    - Design 页面复用首页的 Header 或直接复用 HomeHeader

## 关键实现细节

### 1. 项目创建流程
```
用户输入需求 → 点击生成 → 检查 API Key
    ├── 未设置 → 弹出 ApiKeyModal → 用户填写后继续
    └── 已设置 → 创建新项目(IndexedDB) → 跳转到 /design?projectId=xxx
```

### 2. API Key 检查逻辑
```typescript
const handleGenerate = () => {
  if (!hasApiKey()) {
    setIsApiKeyModalOpen(true);
  }
  // 继续创建项目流程...
};
```

### 3. 项目恢复流程
```
点击项目卡片 → 加载项目数据(IndexedDB) → 跳转到 /design?projectId=xxx → 恢复状态
```

### 4. 状态自动保存
- 设计页面状态变化时，自动同步到 IndexedDB
- 使用防抖保存，避免频繁写入

### 5. 删除确认
- 点击删除按钮，弹出确认对话框
- 确认后从 IndexedDB 删除项目

## 注意事项

1. **组件高内聚**：API Key 弹窗应该组合在 Header 或独立的 ApiKeyModal 组件中，不要在父组件中传递大量 props
2. **样式管理**：全局样式（如滚动条）统一放在 `src/styles/global.css`
3. **代码注释**：遵循项目代码规范，必要时添加中文注释
4. **类型安全**：充分利用 TypeScript 类型系统
5. **Design 页面简化**：模型选择和 API Key 管理迁移到首页后，Design 页面需要相应简化，避免重复代码
6. **LLM 接口复用**：API Key 和模型相关的接口（`getApiKey`、`setApiKey`、`hasApiKey`、`getModel`、`setModel`、`MODEL_LIST`）应保留在 `design/lib/llm.ts` 中，供首页和 Design 页面共同使用
